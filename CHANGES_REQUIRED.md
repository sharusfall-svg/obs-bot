# CHANGES_REQUIRED.md

Список всех параметров, которые **необходимо заполнить** перед запуском.  
Без их заполнения бот запускаться не будет (или будет работать неправильно).

---

## obs_bot.py — обязательные параметры

| Файл | Строка | Переменная | Что вставить | Где взять |
|------|--------|------------|--------------|-----------|
| obs_bot.py | ~35 | `TELEGRAM_TOKEN` | Токен Telegram-бота | @BotFather → /newbot |
| obs_bot.py | ~42 | `ALLOWED_USER_IDS` | Список числовых Telegram ID администраторов | @userinfobot |
| obs_bot.py | ~47 | `BLOCKED_USER_ID` | ID пользователя, которому показывается ложная ошибка (или 0 чтобы отключить) | @userinfobot |
| obs_bot.py | ~52 | `OBS_HOST` | IP-адрес или hostname ПК с OBS | Настройки вашего ПК/роутера |
| obs_bot.py | ~53 | `OBS_PORT` | Порт OBS WebSocket (по умолчанию 4455) | OBS → Инструменты → WebSocket Server |
| obs_bot.py | ~54 | `OBS_PASSWORD` | Пароль OBS WebSocket | OBS → Инструменты → WebSocket Server |
| obs_bot.py | ~58 | `SCENE_MAIN` | Точное имя основной сцены в OBS | OBS → список сцен слева |
| obs_bot.py | ~62 | `SCENE_BREAK` | Точное имя сцены-заглушки в OBS | OBS → список сцен слева |

## Описание сцен OBS

После заполнения параметров убедитесь, что эти сцены существуют в OBS:

| Переменная | Назначение | Пример значения |
|------------|------------|-----------------|
| `SCENE_MAIN` | Основная сцена трансляции — камера, игра, захват экрана. Отображается по кнопке «🌅 Основная сцена» и используется для скриншота. | `"Main"`, `"Трансляция"`, `"Gaming"` |
| `SCENE_BREAK` | Сцена-заглушка / перерыв — статичная картинка или видео «Скоро вернёмся». Отображается по кнопке «💤 Сцена перерыв». | `"BRB"`, `"Перерыв"`, `"Be Right Back"` |

> **Важно:** имена сцен чувствительны к регистру и пробелам. Скопируйте точное название из OBS.

## Как получить нужные данные

### Telegram bot token
1. Написать @BotFather в Telegram
2. Команда `/newbot`
3. Следовать инструкции (имя + username)
4. Скопировать выданный токен вида `1234567890:ABCdefGhIjKlmNoPQRstuVwxYZ`

### Telegram User ID (ALLOWED_USER_IDS, BLOCKED_USER_ID)
1. Написать @userinfobot в Telegram
2. Бот пришлёт ваш числовой ID (например `123456789`)
3. Для нескольких администраторов: `ALLOWED_USER_IDS = [123456789, 987654321]`

### OBS WebSocket (HOST, PORT, PASSWORD)
1. Открыть OBS Studio
2. Меню: **Инструменты → WebSocket Server Settings** (или Tools → WebSocket Server Settings)
3. Включить WebSocket Server
4. Запомнить порт (по умолчанию `4455`) и пароль
5. `OBS_HOST` — IP-адрес компьютера с OBS. Если бот и OBS на одном ПК — `127.0.0.1`

## Порты MediaMTX — что открыть на фаерволе

| Протокол | Порт | Назначение |
|----------|------|------------|
| RTSP | `8554` (UDP/TCP) | Подключение камер, OBS (source) через rtsp:// |
| RTMP | `1935` (TCP) | Публикация потока из OBS через rtmp:// |
| HLS | `8888` (TCP) | Просмотр в браузере через http:// |
| WebRTC | `8889` (TCP) + `8189` (UDP) | WebRTC просмотр/публикация |
| SRT | `8890` (UDP) | SRT-подключение (srt://хост:8890) |

### Строка подключения SRT в OBS

В OBS → Источник → Мультимедиа → Свойства → Убрать галочку локальный файл, поставить галочки только для: перезапустить воспроизведение, ничего не показывать.
Сетевая буферизация - 2 MB
Задержка переподключения - 1 s

```
Ввод - srt://IP_ВАШЕГО_СЕРВЕРА:8890?streamid=read:live/stream

Формат ввода - mpgets
```

### Разрешить IP в mediamtx.yml (для API)

Если нужен удалённый доступ к API/метрикам MediaMTX — добавьте свой IP в `mediamtx.yml`:
```yaml
  ips: ['127.0.0.1', '::1', 'ВАШ_IP']  # IP узнать на 2ip.ru
```

## После заполнения

```bash
# Перезапустить бот
sudo systemctl restart obs_bot

# Проверить логи
journalctl -u obs_bot -f
```

Если бот запустился — напишите ему `/start` в Telegram.
